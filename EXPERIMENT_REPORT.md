# 计算机网络实验二：DNS 中继服务器

## 1. 功能设计

程序实现一个 UDP DNS 中继服务器。启动后读取 `dnsrelay.txt` 中的 `IP 域名` 对照表，并监听本地 DNS 查询。

查询处理分为三类：

1. 表中命中普通 IPv4 地址：直接构造 DNS A 记录响应并返回客户端。
2. 表中命中 `0.0.0.0`：返回 NXDOMAIN，用于不良网站拦截。
3. 表中未命中：修改 DNS 报文 ID 后转发到外部 DNS 服务器，收到响应后恢复客户端原始 ID 并返回。

## 2. 模块划分

- 参数解析：处理 `-d`、`-dd`、`-p`、上游 DNS 地址和配置文件路径。
- 域名表加载：逐行读取 `IP domain`，域名统一转为小写并去掉末尾点。
- DNS 报文解析：读取 Header、Question、QNAME、QTYPE、QCLASS。
- 本地响应构造：对 A 查询生成普通 A 记录或 NXDOMAIN 响应。
- 中继与 ID 转换：用新 ID 替换客户端 ID，保存原始 ID、客户端地址和时间戳。
- 超时清理：周期性删除长时间未收到上游响应的 ID 映射。

## 3. 软件流程

```text
启动
  |
  +-- 解析命令行参数
  +-- 加载 dnsrelay.txt
  +-- 创建并绑定 UDP socket
  |
循环 select 等待 UDP 报文
  |
  +-- 来自上游 DNS 的响应？
  |      |
  |      +-- 是：按 ID 找到映射，恢复原 ID，发回客户端
  |
  +-- 来自客户端的查询
         |
         +-- 解析域名和查询类型
         +-- 命中 0.0.0.0：返回 NXDOMAIN
         +-- 命中普通 IP：返回 A 记录
         +-- 未命中：分配中继 ID，转发上游 DNS
```

## 4. 测试用例

使用非 53 端口便于普通权限测试：

```powershell
.\build\dnsrelay.exe -dd -p 10530 8.8.8.8 .\dnsrelay.txt
```

| 用例 | 命令 | 预期 |
| --- | --- | --- |
| 本地解析 | `nslookup -port=10530 test1 127.0.0.1` | 返回 `11.111.11.111` |
| 拦截域名 | `nslookup -port=10530 test0 127.0.0.1` | 返回 NXDOMAIN |
| 中继解析 | `nslookup -port=10530 www.bupt.edu.cn 127.0.0.1` | 由上游 DNS 返回解析结果 |

## 5. 调试问题

- 端口 53 在 Windows 上可能需要管理员权限，因此实现中增加了 `-p <port>` 选项用于实验验证。
- DNS 报文中的 ID 不能直接透传给上游，否则并发请求可能混淆。本实现为每个中继请求分配新 ID，并保存映射。
- 域名比较需要大小写无关，且 DNS 查询域名可能带末尾点，因此加载和查询时统一规范化。

## 6. 总结

本实验完成了 DNS 查询报文解析、UDP socket 收发、本地 DNS 响应构造、外部 DNS 中继、ID 转换和超时清理，覆盖了 DNS 中继服务器的主要实验要求。
